# 0.3.4 Embedded Workflow Prompts Design

## Goal

Add deterministic scanning for AI-agent instructions embedded inside GitHub Actions workflows, starting with Claude Code Action prompt-related inputs. These prompts are agent-visible context, but today they are hidden from `drctx manifest`, `drctx check`, and context hygiene findings.

## Problem

Repositories increasingly put agent instructions in CI workflow YAML instead of checked-in instruction files. For Claude Code Action this commonly appears in `with:` inputs such as:

- `custom_instructions`
- `claude_args` containing `--append-system-prompt`
- `prompt`
- `direct_prompt`

Those values can change what an agent sees without showing up in AGENTS, CLAUDE, Cursor rules, or manifest effective context. This creates a context rot gap: CI may instruct the agent to skip tests, override repo rules, or follow stale hidden guidance while local developers see a clean context contract.

## Constraints

- Stay deterministic, local, and read-only.
- No network calls, LLM calls, or provider API calls.
- Treat workflow prompt extraction as source-backed facts with file, line, and text evidence.
- Avoid noisy semantic claims. 0.3.4 should flag only explicit, source-backed risks.
- Keep checks pure. Extractors parse workflow files; checks consume `CheckContext` only.
- Do not add workflow execution, shell parsing beyond conservative flag extraction, or child config inheritance.
- Public tests and docs must use synthetic workflow examples only.

## Design

### Data model

Add a new fact type:

```ts
export type WorkflowPromptKind =
  | 'custom-instructions'
  | 'append-system-prompt'
  | 'prompt'
  | 'direct-prompt';

export type WorkflowPrompt = {
  kind: WorkflowPromptKind;
  action: string;
  value: string;
  source: SourceSpan;
};
```

Add `workflowPrompts: WorkflowPrompt[]` to `RepoFacts` and `Manifest`.

Manifest JSON should expose workflow prompts as a first-class section:

```json
{
  "workflowPrompts": [
    {
      "kind": "custom-instructions",
      "action": "anthropics/claude-code-action@v1",
      "value": "Always run pnpm test before committing.",
      "source": {
        "file": ".github/workflows/agent.yml",
        "line": 8,
        "text": "custom_instructions: Always run pnpm test before committing."
      }
    }
  ],
  "summary": {
    "workflowPrompts": 1
  }
}
```

Text manifest should add a short `Workflow prompts` section only when prompts exist.

### Extractor

Create `src/extractors/workflowPrompts.ts`.

It should scan only root-relative GitHub workflow files matching:

```text
.github/workflows/*.yml
.github/workflows/*.yaml
```

It should parse YAML with the existing `yaml` dependency, then walk `jobs.*.steps[]` and identify steps with a `uses` value that looks like a Claude Code Action. For 0.3.4, match action names conservatively:

```text
anthropics/claude-code-action
anthropics/claude-code-base-action
```

Version suffixes such as `@v1`, `@beta`, or commit SHAs should be preserved in the `action` field.

Extract prompt values from `step.with` keys:

- `custom_instructions` -> `custom-instructions`
- `prompt` -> `prompt`
- `direct_prompt` -> `direct-prompt`
- `claude_args` -> parse only `--append-system-prompt <value>` and `--append-system-prompt=<value>`

For `claude_args`, extraction must handle simple quoted strings and block scalars. It does not need a full shell parser. If the extractor cannot confidently isolate a value, it should skip rather than guess.

### Source locations

Use line-based source discovery similar to `ciCommands.ts`:

- exact line for direct `custom_instructions`, `prompt`, and `direct_prompt` keys;
- first line containing `--append-system-prompt` for `claude_args`;
- `text` should be the trimmed line, not the whole block.

The YAML parsed value is the canonical `value`; the source text is evidence only.

### Checks

Add two conservative checks.

#### `unsafe-workflow-prompt`

Flag workflow prompts that contain explicit bypass guidance already recognized by `unsafeAgentInstructionsCheck`:

- `skip tests`
- `ignore lint`
- `--no-verify`
- `force push`
- `git push --force`

Use the same negation guard behavior as agent instruction checks so `never skip tests` does not flag.

Severity: `warning`.

Confidence: `medium`.

Primary source: workflow prompt source span.

Suggestion: move safe instructions into repo-visible agent docs, or replace bypass guidance with explicit verification expectations.

#### `hidden-workflow-prompt`

Flag when workflow prompts exist and there are no repo-visible agent instruction files.

This is not an error. It says: "the only agent instructions Dr. Context found are embedded in CI." That is actionable because humans and local agents may miss them.

Severity: `info`.

Confidence: `high`.

Primary source: first workflow prompt source span.

Suggestion: add or reference canonical agent instructions in `AGENTS.md` or `CLAUDE.md`.

Do not flag this when at least one repo/path/nested agent instruction document exists. Workflow prompts can then be represented as an additional surface, not the sole source of truth.

## CLI and reporting behavior

No new CLI flags.

Existing commands should gain the new facts automatically:

- `drctx check`: reports new findings.
- `drctx manifest`: lists workflow prompts.
- `drctx check --json`: includes workflow-prompt findings when present.
- `drctx check --sarif`: includes workflow-prompt findings through existing SARIF rendering.
- `drctx check --workspace`: each candidate scan handles its own workflow prompts.

Baseline/config suppression from 0.3.3 works unchanged because findings use normal IDs and source spans.

## Test strategy

Add focused tests rather than large fixtures:

- extractor unit tests for direct `custom_instructions`, `prompt`, `direct_prompt`, and `claude_args --append-system-prompt` values;
- negative extractor tests for non-Claude actions and non-workflow YAML files;
- check tests for unsafe workflow prompt and hidden workflow prompt;
- manifest tests for `workflowPrompts` JSON/text output;
- CLI smoke test showing baseline suppression can suppress a workflow prompt finding, if this stays cheap.

The clean fixture must still produce zero findings.

## Rollout

0.3.4 should be a minor additive release:

- No breaking schema change beyond adding optional `workflowPrompts` and `summary.workflowPrompts` to manifest JSON.
- No new required config.
- Findings are conservative: one warning for explicit unsafe prompt content, one info for hidden prompt-only context.

## Deferred

- Prompt conflict semantic analysis.
- Workflow prompt extraction for non-Claude providers.
- Full shell parsing of `claude_args`.
- GitHub Action marketplace API lookup.
- MCP context gate.
- AI remediation.
