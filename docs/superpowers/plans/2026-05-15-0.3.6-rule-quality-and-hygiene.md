# 0.3.6 Rule Quality and Hygiene Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add low-noise deterministic rule-quality and safety/workflow hygiene checks after 0.3.5 drift checks ship.

**Architecture:** Implement pure checks over already extracted instruction and repo-policy facts, adding deterministic helpers only where required. Findings should start as `info` unless repo evidence is strong enough for `warning`.

**Tech Stack:** TypeScript, Vitest, existing scanner pipeline, existing reporters.

## 0.3.6 scope decision

0.3.6 uses a conservative safety-first scope:

- Ship policy visibility checks:
  - `hidden-secret-hygiene-policy`
  - `hidden-destructive-action-policy`
  - `missing-generated-file-boundary`
  - `hidden-workflow-policy`
- Ship only exact deterministic rule-quality checks:
  - `oversized-instruction-file`
  - `duplicate-instruction-block`
- Defer broader heuristic rule-quality checks until later pre-0.4 work:
  - `low-specificity-instruction`
  - `copied-docs-in-instructions`
  - `missing-instruction-example`

Deferred checks are intentionally retained in this plan so they are not lost. Revisit them during 0.3.7 dogfood/examples or 0.3.8 context-health readiness after 0.3.6 noise is measured.

## Candidate tasks

### Task 1: Finalize deterministic rule-quality thresholds

- [x] Decide instruction-size thresholds by surface type.
- [x] Decide duplicate block minimum length and normalization rules.
- [x] Defer low-specificity phrase list and neighboring concrete-evidence guard until later pre-0.4 work.
- [x] Commit the decisions to this plan or a 0.3.6 design note before coding.

### Task 2: Oversized and duplicated instruction checks

- [ ] Add failing tests for oversized instruction files.
  - [ ] `AGENTS.md` over 500 lines emits `oversized-instruction-file`.
  - [ ] `AGENTS.md` over 30 KB emits `oversized-instruction-file`.
  - [ ] Cursor rule over 500 lines emits `oversized-instruction-file`.
  - [ ] Below-threshold instruction files do not emit.
- [ ] Add failing tests for exact or normalized duplicate instruction blocks.
  - [ ] Exact duplicate block with at least 5 non-empty lines emits `duplicate-instruction-block`.
  - [ ] Normalized whitespace duplicate with at least 300 characters emits `duplicate-instruction-block`.
  - [ ] Frontmatter-only overlap does not emit.
  - [ ] Short boilerplate overlap does not emit.
- [ ] Implement pure checks and deterministic helpers.
  - [ ] Use normalized block fingerprints in a map; do not perform pairwise fuzzy comparisons.
- [ ] Verify targeted tests pass.
- [ ] Commit.

### Deferred pre-0.4 backlog: Low-specificity and copied-doc checks

Deferred until later pre-0.4 work. Do not implement in 0.3.6 unless the user explicitly expands scope after dogfood data.

- Future check: `low-specificity-instruction`.
  - Add failing tests for vague guidance without concrete examples.
  - Add no-finding tests when vague phrases are followed by concrete commands, paths, or examples.
  - Start as `info` only.
- Future check: `copied-docs-in-instructions`.
  - Add copied-doc positive and negative fixtures.
  - Prefer exact normalized block overlap, not semantic similarity.
  - Suggest linking to canonical docs instead of duplicating long text.

### Deferred pre-0.4 backlog: Missing canonical example check

Deferred until later pre-0.4 work. Do not implement in 0.3.6 unless the user explicitly expands scope after dogfood data.

- Future check: `missing-instruction-example`.
  - Add failing tests for instructions that require following a style or pattern without naming an example file.
  - Add no-finding tests for instructions that provide concrete examples.
  - Suppress when nearby text names a path, glob, symbol, command, or example file.

### Task 5: Safety/workflow hygiene checks

- [ ] Decide which repo docs count as canonical policy sources.
  - [ ] Add narrow read globs for `SECURITY.md`, `CONTRIBUTING.md`, `docs/SECURITY.md`, `docs/CONTRIBUTING.md`, `.github/pull_request_template.md`, and `.github/ISSUE_TEMPLATE/*.{md,yml,yaml}` if not already read.
  - [ ] Do not add broad recursive `docs/**/*.md` discovery.
- [ ] Add tests for hidden secret hygiene policy.
  - [ ] Strong canonical policy plus missing agent-visible secret guidance emits.
  - [ ] Agent-visible secret guidance suppresses.
  - [ ] No canonical policy source does not emit.
  - [ ] Weak generic words such as `secure` alone do not emit.
- [ ] Add tests for hidden destructive-action policy.
  - [ ] Explicit destructive-action policy plus missing agent-visible boundary emits.
  - [ ] Agent-visible destructive-action boundary suppresses.
  - [ ] Weak generic caution wording does not emit.
- [ ] Add tests for missing generated/dist do-not-edit boundary.
  - [ ] `package.json` `files`, `main`, `types`, or `bin` pointing at generated output plus missing agent-visible boundary emits.
  - [ ] Agent-visible generated/dist boundary suppresses.
  - [ ] Ignored `dist/` or `build/` output alone, without package metadata evidence, does not emit.
- [ ] Add tests for hidden planning/TDD/review/verification policy.
  - [ ] Canonical workflow policy plus missing agent-visible workflow guidance emits.
  - [ ] Agent-visible workflow guidance suppresses.
  - [ ] Generic docs process wording without concrete policy does not emit.
- [ ] Implement pure checks.
- [ ] Verify targeted tests pass.
- [ ] Commit.

## Guardrails

- Prefer false negatives over noisy false positives.
- Use `info` by default.
- Do not add semantic LLM judgments.
- Do not flag missing policies unless repo evidence shows the policy exists outside agent-visible docs.
