# 0.3.6 Rule Quality and Hygiene Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add low-noise deterministic rule-quality and safety/workflow hygiene checks after 0.3.5 drift checks ship.

**Architecture:** Implement pure checks over already extracted instruction and repo-policy facts, adding deterministic helpers only where required. Findings should start as `info` unless repo evidence is strong enough for `warning`.

**Tech Stack:** TypeScript, Vitest, existing scanner pipeline, existing reporters.

## Candidate tasks

### Task 1: Finalize deterministic rule-quality thresholds

- [ ] Decide instruction-size thresholds by surface type.
- [ ] Decide duplicate block minimum length and normalization rules.
- [ ] Decide low-specificity phrase list and neighboring concrete-evidence guard.
- [ ] Commit the decisions to this plan or a 0.3.6 design note before coding.

### Task 2: Oversized and duplicated instruction checks

- [ ] Add failing tests for oversized instruction files.
- [ ] Add failing tests for exact or normalized duplicate instruction blocks.
- [ ] Implement pure checks and deterministic helpers.
- [ ] Verify targeted tests pass.
- [ ] Commit.

### Task 3: Low-specificity and copied-doc checks

- [ ] Add failing tests for vague guidance without concrete examples.
- [ ] Add no-finding tests when vague phrases are followed by concrete commands, paths, or examples.
- [ ] Add copied-doc positive and negative fixtures.
- [ ] Implement pure checks.
- [ ] Verify targeted tests pass.
- [ ] Commit.

### Task 4: Missing canonical example check

- [ ] Add failing tests for instructions that require following a style or pattern without naming an example file.
- [ ] Add no-finding tests for instructions that provide concrete examples.
- [ ] Implement pure check.
- [ ] Verify targeted tests pass.
- [ ] Commit.

### Task 5: Safety/workflow hygiene checks

- [ ] Decide which repo docs count as canonical policy sources.
- [ ] Add tests for hidden secret hygiene policy.
- [ ] Add tests for hidden destructive-action policy.
- [ ] Add tests for missing generated/dist do-not-edit boundary.
- [ ] Add tests for hidden planning/TDD/review/verification policy.
- [ ] Implement pure checks.
- [ ] Verify targeted tests pass.
- [ ] Commit.

### Task 6: Docs, review, and release prep

- [ ] Update README, CHANGELOG, ARCHITECTURE, roadmap, and release docs.
- [ ] Run full verification gate from the umbrella spec.
- [ ] Run independent review.
- [ ] Release only if Dr. Context self-scan remains clean or accepted findings are documented.

## Guardrails

- Prefer false negatives over noisy false positives.
- Use `info` by default.
- Do not add semantic LLM judgments.
- Do not flag missing policies unless repo evidence shows the policy exists outside agent-visible docs.
