## Immediate next step

Run published-package dogfood after every release that materially changes scanner behavior. Use aggregate public-safe output only.

## Recommended order

1. Surface coverage refresh and docs cleanup.
2. `drctx init` because it is the safest dry-run-first write-mode candidate.
3. Ruler and Repomix docs recipes because they test demand without dependencies.
4. Child config inheritance if monorepo dogfood shows confusion.
5. MCP context gate because it reuses deterministic scan and manifest outputs.
6. Safe fixes because patch output needs stronger safety confidence.
7. Direct cross-agent drift after more dogfood calibration.
8. Agent probes last because provider runtime claims are hardest to verify and support.

## Rule

Each production feature needs its own implementation plan before code changes.
