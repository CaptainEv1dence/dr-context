## Task 1: Lock current Action contract in tests

**Files:**
- Modify: `tests/githubAction.test.ts`

- [ ] **Step 1: Add current-contract tests**

Add tests to `tests/githubAction.test.ts`:

```ts
import { readFileSync } from 'node:fs';
import { describe, expect, it } from 'vitest';

describe('GitHub Action contract', () => {
  const action = readFileSync('action.yml', 'utf8');

  it('keeps stable public inputs', () => {
    for (const input of ['root', 'strict', 'upload-sarif', 'version']) {
      expect(action).toContain(`${input}:`);
    }
    expect(action).not.toContain('sarif:');
  });

  it('runs the selected npm package version through npm exec', () => {
    expect(action).toContain('DR_CONTEXT_VERSION: ${{ inputs.version }}');
    expect(action).toContain('--package "dr-context@$DR_CONTEXT_VERSION"');
    expect(action).toContain('dr-context check --sarif --root "$DR_CONTEXT_ROOT"');
  });

  it('preserves SARIF upload gating', () => {
    expect(action).toContain("inputs.upload-sarif == 'true'");
    expect(action).toContain('steps.scan.outputs.sarif_ready ==');
    expect(action).toContain('github/codeql-action/upload-sarif@v4');
  });
});
```

In `docs/github-action.md`, add:

```md
## Reliability notes

The current Action wrapper is a composite Action. It installs the selected `dr-context` npm package with `npm exec` before running the scan.

This means:

- pinning `uses: CaptainEv1dence/dr-context@v0.3.9` pins the Action wrapper;
- `version` selects the npm package version the wrapper executes;
- CI needs npm registry access before scanning;
- the scan itself remains local, deterministic, read-only, and does not call LLMs or external services.

Bundled Action work is tracked separately. It must preserve the current inputs, SARIF output, annotations, summary, and exit-code behavior before replacing the composite wrapper.
```
