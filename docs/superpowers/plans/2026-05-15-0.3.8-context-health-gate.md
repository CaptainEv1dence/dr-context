# 0.3.8 Context Health Summary Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a deterministic, trend-friendly context health summary to scan and workspace reports without making the score a hard gate.

**Architecture:** Findings remain the product; health is a pure summary derived from visible, unsuppressed finding counts after suppression filtering. A small scoring helper owns the formula, `summary` embeds the resulting health object, and reporters only render/report it without inferring new facts.

**Tech Stack:** TypeScript, Vitest, existing Dr. Context scan/reporting pipeline.

---

## Source spec and decision

Source umbrella spec:

```text
docs/superpowers/specs/2026-05-15-pre-0.4-context-quality-design.md
```

Decision for `0.3.8`: ship **Option A**, a deterministic JSON/text health summary only.

Do **not** ship in `0.3.8`:

- badge generation;
- `--min-health` or any score-based exit-code flag;
- persisted health history, score trends, or snapshots;
- hosted score services;
- telemetry-backed benchmarks;
- AI-generated health summaries.

Compatibility decisions:

- Keep `drctx.report.v1` and `drctx.workspace-report.v1`; `summary.health` is an additive summary field.
- Findings, fingerprints, SARIF results, and baseline files remain source-of-truth outputs. Health does not change finding identity.
- Baseline matching, fail-on-new behavior, and suppression matching ignore health entirely.
- `summary.health` is not the context manifest or future context contract. `drctx manifest`, MCP gate, agent probes, and first-read contracts remain separate roadmap layers.
- Consumers should tolerate additive fields under `summary`; strict consumers can read `errors`, `warnings`, and `infos` exactly as before.

---

## Locked scoring formula

Health is derived from visible findings after suppressions are applied:

```text
score = clamp(100 - errors * 35 - warnings * 10 - infos * 2, 0, 100)
```

Grades:

- `excellent`: `95..100`
- `good`: `80..94`
- `fair`: `60..79`
- `poor`: `0..59`

Suppressed findings do not reduce score. The health object still includes `suppressedCount` so accepted debt is visible.

Use this TypeScript shape:

```ts
export type HealthGrade = 'excellent' | 'good' | 'fair' | 'poor';

export type HealthSummary = {
  score: number;
  grade: HealthGrade;
  penalties: {
    errors: number;
    warnings: number;
    infos: number;
  };
  suppressedCount: number;
};
```

- [ ] **Step 4: Implement pure helper**

Create `src/core/health.ts`:

```ts
import type { HealthGrade, HealthSummary, Report } from './types.js';

type FindingCounts = {
  errors: number;
  warnings: number;
  infos: number;
  suppressed?: number;
};

const ERROR_PENALTY = 35;
const WARNING_PENALTY = 10;
const INFO_PENALTY = 2;

export function calculateHealthSummary(counts: FindingCounts): HealthSummary {
  const penalties = {
    errors: counts.errors * ERROR_PENALTY,
    warnings: counts.warnings * WARNING_PENALTY,
    infos: counts.infos * INFO_PENALTY
  };
  const score = clampScore(100 - penalties.errors - penalties.warnings - penalties.infos);

  return {
    score,
    grade: gradeForScore(score),
    penalties,
    suppressedCount: counts.suppressed ?? 0
  };
}

export function healthSummaryFromReports(reports: Report[]): HealthSummary {
  return calculateHealthSummary({
    errors: reports.reduce((total, report) => total + report.summary.errors, 0),
    warnings: reports.reduce((total, report) => total + report.summary.warnings, 0),
    infos: reports.reduce((total, report) => total + report.summary.infos, 0),
    suppressed: reports.reduce((total, report) => total + (report.summary.suppressed ?? 0), 0)
  });
}

function clampScore(score: number): number {
  return Math.max(0, Math.min(100, score));
}

function gradeForScore(score: number): HealthGrade {
  if (score >= 95) {
    return 'excellent';
  }
  if (score >= 80) {
    return 'good';
  }
  if (score >= 60) {
    return 'fair';
  }
  return 'poor';
}
```
